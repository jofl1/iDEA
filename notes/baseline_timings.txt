iDEA interacting solver — Phase A baseline measurements

Captured before the determinant-basis matrix-free LinearOperator work
(notes/friend_optimization_advice.md). These numbers are the reference
point for Phase B speedup claims.

Hardware: user's Mac (Darwin 24.6.0).
Solver: current iDEA.methods.interacting.solve (CSR labelled tensor basis +
  scipy.sparse.linalg.eigsh + post-solve antisymmetrize).

Cases:
  interacting_uu_80   — 1↑1↓ harmonic, N=80, electrons=2
  interacting_uud_60  — 2↑1↓ harmonic, N=60, electrons=3

  The 4-electron case (uudd) is deferred to Phase B. At all 4-electron
  sizes the current solver's antisymmetrize() pass stochastically fails
  with IndexError when all Lanczos eigenvectors antisymmetrize to zero
  (a manifestation of the wrong-basis problem in the friend's review).
  The determinant-basis switch eliminates the failure mode entirely.

Re-measure with:
  python benchmarks/benchmark_interacting.py time --runs 3

Generate baselines for cross-machine sanity checks with:
  python benchmarks/benchmark_interacting.py generate

Validate current solver against committed baselines with:
  python benchmarks/benchmark_interacting.py compare
